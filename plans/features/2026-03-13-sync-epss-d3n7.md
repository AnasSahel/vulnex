# EPSS Sync Worker

## Description

Fetch EPSS scores from the FIRST.org API and update the `cve_score` table with exploit probability data.

### Current problems

1. EPSS scores in the dashboard are hardcoded.
2. No way to track EPSS score changes over time.

### Target design

**Motia step: `sync-epss.step.ts`**

- Subscribes to: `sync.epss` topic
- API: `https://api.first.org/data/v1/epss`
- Supports bulk queries: `?cve=CVE-2024-3094,CVE-2024-4577,...`
- Strategy: fetch EPSS for all CVEs currently in the `cve` table

**Data mapping:**
- `cve_score.epss_score` ← `data[].epss`
- `cve_score.epss_percentile` ← `data[].percentile`
- `cve_score.source` = `"epss"`
- `cve_score.scored_at` ← `data[].date`

**Batch processing:**
- Query the `cve` table for all CVE IDs
- Batch requests to FIRST API (100 CVEs per request)
- Upsert scores into `cve_score`

## User Stories

1. As a user, I want to see real EPSS scores so I can prioritize by exploit likelihood.
2. As the system, I need to keep EPSS scores current for all tracked CVEs.

- If no CVEs exist in DB yet, skip gracefully
