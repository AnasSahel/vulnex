# GHSA Sync Worker

## Description

Fetch security advisories from the GitHub Advisory Database via the GraphQL API and store them in the `advisory` table.

### Current problems

1. No advisory data in the dashboard.
2. Can't show which CVEs have published GitHub security advisories.

### Target design

**Motia step: `sync-ghsa.step.ts`**

- Subscribes to: `sync.ghsa` topic
- API: GitHub GraphQL API (`https://api.github.com/graphql`)
- Authentication: requires `GITHUB_TOKEN` env var (public read access)
- Pagination: cursor-based via GraphQL `after` parameter

**GraphQL query:**
```graphql
query($after: String) {
  securityAdvisories(first: 100, after: $after, orderBy: { field: UPDATED_AT, direction: DESC }) {
    nodes {
      ghsaId
      summary
      description
      severity
      publishedAt
      updatedAt
      permalink
      identifiers { type value }
    }
    pageInfo { hasNextPage endCursor }
  }
}
```

**Data mapping:**
- `advisory.ghsa_id` ← `ghsaId`
- `advisory.cve_id` ← `identifiers.find(i => i.type === "CVE")?.value`
- `advisory.severity` ← `severity` (CRITICAL, HIGH, MODERATE, LOW)
- `advisory.summary` ← `summary`
- `advisory.description` ← `description`
- `advisory.published_at` ← `publishedAt`
- `advisory.url` ← `permalink`

**Incremental sync:**
- Store last `endCursor` in `sync_log.last_cursor`
- On subsequent runs, use `updatedSince` filter or cursor

## User Stories

1. As a user, I want to see GitHub advisories linked to my watched CVEs.
2. As the system, I need to track new advisories for the alerts feed.
