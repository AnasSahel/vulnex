## Item 9: Scanner Output Ingestion (`vulnex prioritize`)

The single most important feature for positioning. Users keep their scanner, add vulnex for the "so what?" layer.

### Usage

```bash
# Pipe from scanner
trivy fs . -f json | vulnex prioritize
grype . -o json | vulnex prioritize

# From file
vulnex prioritize --from trivy scan-results.json
vulnex prioritize --from grype grype-output.json
vulnex prioritize --from sarif results.sarif

# Auto-detect format
vulnex prioritize results.json

# Output control
vulnex prioritize --from trivy scan.json -o table
vulnex prioritize --from trivy scan.json -o json
vulnex prioritize --from trivy scan.json -o sarif
vulnex prioritize --from trivy scan.json --scoring-profile exploit-focused
```

### Input Formats

**Trivy JSON** (`--from trivy`):
```json
{
  "SchemaVersion": 2,
  "Results": [{
    "Target": "path/to/lockfile",
    "Type": "gomod",
    "Vulnerabilities": [{
      "VulnerabilityID": "CVE-2024-24790",
      "PkgName": "stdlib",
      "InstalledVersion": "1.21.0",
      "FixedVersion": "1.21.11",
      "Severity": "CRITICAL",
      "Title": "...",
      "Description": "..."
    }]
  }]
}
```

**Grype JSON** (`--from grype`):
```json
{
  "matches": [{
    "vulnerability": {
      "id": "CVE-2024-24790",
      "severity": "Critical",
      "fix": { "state": "fixed" }
    },
    "artifact": {
      "name": "stdlib",
      "version": "1.21.0",
      "type": "go-module"
    }
  }]
}
```

**SARIF** (`--from sarif`):
```json
{
  "$schema": "https://raw.githubusercontent.com/oasis-tcs/sarif-spec/main/sarif-2.1/schema/sarif-schema-2.1.0.json",
  "version": "2.1.0",
  "runs": [{
    "tool": { "driver": { "name": "...", "rules": [...] } },
    "results": [{ "ruleId": "CVE-...", "level": "error", ... }]
  }]
}
```

### Implementation

#### New files

**`internal/scanner/scanner.go`** ‚Äî Scanner finding types and parser interface:
```go
package scanner

// Finding represents a normalized vulnerability finding from any scanner.
type Finding struct {
    CVE       string // CVE ID (primary key for enrichment)
    Package   string // Package/component name
    Version   string // Installed version
    Fixed     string // Fixed version (empty if unknown)
    Severity  string // Original severity from scanner
    Ecosystem string // npm, go, pypi, maven, etc.
    Source    string // "trivy", "grype", "sarif"
    Target   string // File/image that was scanned
}

// Parser parses scanner output into normalized findings.
type Parser interface {
    Parse(r io.Reader) ([]Finding, error)
}
```

**`internal/scanner/trivy.go`** ‚Äî Trivy JSON parser:
- Parse `Results[].Vulnerabilities[]`
- Map `VulnerabilityID` ‚Üí `CVE`, `PkgName` ‚Üí `Package`, `InstalledVersion` ‚Üí `Version`
- Map Trivy `Type` to ecosystem: `gomod`‚ÜíGo, `npm`‚Üínpm, `pip`‚ÜíPyPI, `cargo`‚Üícrates.io, etc.

**`internal/scanner/grype.go`** ‚Äî Grype JSON parser:
- Parse `matches[].vulnerability` + `matches[].artifact`
- Map `vulnerability.id` ‚Üí `CVE`, `artifact.name` ‚Üí `Package`, `artifact.version` ‚Üí `Version`
- Map Grype `artifact.type` to ecosystem: `go-module`‚ÜíGo, `npm`‚Üínpm, `python`‚ÜíPyPI, etc.

**`internal/scanner/sarif.go`** ‚Äî SARIF input parser:
- Parse `runs[].results[]` ‚Äî extract `ruleId` as CVE ID
- Extract package/version from `message.text` or `properties` if available
- Graceful degradation: if only CVE IDs available, enrich without package context

**`internal/scanner/detect.go`** ‚Äî Auto-detection:
- JSON with `SchemaVersion` + `Results` ‚Üí Trivy
- JSON with `matches` + `descriptor` ‚Üí Grype
- JSON with `$schema` containing "sarif" ‚Üí SARIF
- Return `(Parser, string)` where string is the detected format name

**`cmd/prioritize.go`** ‚Äî The command:
```go
var prioritizeCmd = &cobra.Command{
    Use:   "prioritize [file]",
    Short: "Enrich and prioritize scanner findings",
    Long: `Ingest vulnerability findings from Trivy, Grype, or SARIF and enrich
each finding with EPSS scores, KEV status, exploit availability, and
composite risk priority (P0-P4).

Accepts input from stdin (pipe) or from a file argument.`,
    Example: `  trivy fs . -f json | vulnex prioritize
  vulnex prioritize --from trivy scan-results.json
  vulnex prioritize --from grype output.json -o table
  vulnex prioritize results.sarif --scoring-profile exploit-focused`,
    Args: cobra.MaximumNArgs(1),
    RunE: runPrioritize,
}
```

#### Pipeline

1. Read input (stdin or file arg)
2. Detect or use `--from` flag to select parser
3. Parse ‚Üí `[]scanner.Finding`
4. Deduplicate by CVE ID (keep all package associations)
5. Collect unique CVE IDs
6. `app.Enricher.EnrichBatch(ctx, cveIDs)` ‚Üí enrich with EPSS/KEV/NVD/GHSA/OSV
7. `app.Exploit.CheckBatch(ctx, cveIDs)` ‚Üí exploit availability
8. Convert to `[]model.SBOMFinding` (reuse existing type + enrichment fields)
9. Apply `model.ComputeRisk()` per finding
10. Sort by risk priority (P0 first, then by composite score desc)
11. Apply suppressions (`.vulnexignore`)
12. Output via `app.Formatter.FormatSBOMResult()`

#### New model type: `PrioritizeResult`

```go
type PrioritizeResult struct {
    Source     string        `json:"source"`     // "trivy", "grype", "sarif"
    InputFile  string        `json:"input_file"` // file path or "stdin"
    Findings   []SBOMFinding `json:"findings"`
    Suppressed []SBOMFinding `json:"suppressed,omitempty"`
    Summary    PrioritizeSummary `json:"summary"`
}

type PrioritizeSummary struct {
    Total     int            `json:"total"`
    ByPriority map[string]int `json:"by_priority"` // P0: 2, P1: 5, ...
    Enriched  int            `json:"enriched"`
    WithExploit int          `json:"with_exploit"`
}
```

Add `FormatPrioritizeResult` to the Formatter interface, or reuse `FormatSBOMResult` since findings share the same type.

#### Flags

```
--from        Scanner format: trivy, grype, sarif (auto-detected if omitted)
--ignore-file Path to .vulnexignore suppression file
--strict      Ignore suppression file
--scoring-profile  Scoring profile name or path to custom profile
```

## Item 10: Policy-as-Code (YAML Rules)

Replace the blunt `--severity` flag with composable rules for CI gates.

### Usage

```bash
# Use a policy file
vulnex prioritize scan.json --policy policy.yaml
vulnex scan go.sum --enrich --policy policy.yaml

# Inline quick rules
vulnex prioritize scan.json --fail-on "kev=true"
vulnex prioritize scan.json --fail-on "priority=P0,P1"
```

### Policy File Format

```yaml
# .vulnex-policy.yaml
version: 1
rules:
  - name: "Block KEV vulnerabilities"
    match:
      kev: true
    action: fail

  - name: "Block critical with high exploit probability"
    match:
      epss_gte: 0.7
      severity: [critical, high]
    action: fail

  - name: "Warn on available exploits"
    match:
      has_exploit: true
    action: warn

  - name: "Allow medium in dev dependencies"
    match:
      severity: [medium, low]
      ecosystem: [npm]  # optional scope
    action: allow
```

### Implementation

**`internal/policy/policy.go`** ‚Äî Policy engine:
```go
type Policy struct {
    Version int    `yaml:"version"`
    Rules   []Rule `yaml:"rules"`
}

type Rule struct {
    Name   string    `yaml:"name"`
    Match  Condition `yaml:"match"`
    Action string    `yaml:"action"` // "fail", "warn", "allow"
}

type Condition struct {
    KEV        *bool    `yaml:"kev,omitempty"`
    Severity   []string `yaml:"severity,omitempty"`
    EPSSGte    *float64 `yaml:"epss_gte,omitempty"`
    EPSSLte    *float64 `yaml:"epss_lte,omitempty"`
    CVSSGte    *float64 `yaml:"cvss_gte,omitempty"`
    HasExploit *bool    `yaml:"has_exploit,omitempty"`
    Priority   []string `yaml:"priority,omitempty"`     // P0, P1, P2, etc.
    Ecosystem  []string `yaml:"ecosystem,omitempty"`
}

// Evaluate returns the action for a finding: "fail", "warn", "allow", or "" (no match).
func (p *Policy) Evaluate(f model.SBOMFinding) string

// EvaluateAll returns findings grouped by action.
func (p *Policy) EvaluateAll(findings []model.SBOMFinding) PolicyResult
```

**`internal/policy/result.go`** ‚Äî Policy evaluation result:
```go
type PolicyResult struct {
    Failures []PolicyViolation `json:"failures"`
    Warnings []PolicyViolation `json:"warnings"`
    Allowed  []PolicyViolation `json:"allowed"`
    Passed   bool              `json:"passed"` // true if no failures
}

type PolicyViolation struct {
    Finding  model.SBOMFinding `json:"finding"`
    RuleName string            `json:"rule_name"`
    Action   string            `json:"action"`
}
```

**Integration into commands:**
- `cmd/prioritize.go`: after enrichment, evaluate policy ‚Üí exit 1 if failures
- `cmd/scan.go`: when `--policy` is set, use policy instead of bare `os.Exit(1)`
- Policy replaces the current "exit 1 if any findings" logic with configurable rules

**Default policy location:** `.vulnex-policy.yaml` in project root (like `.vulnexignore`)

## Item 11: Temporal Intelligence

Surface EPSS trend data in enrichment to answer "is this getting more dangerous?"

### Implementation

**Extend `SBOMFinding`** (or `PrioritizeResult` findings):
```go
EPSSTrend  *EPSSTrend `json:"epss_trend,omitempty"`
```

```go
type EPSSTrend struct {
    Current    float64 `json:"current"`
    Previous30 float64 `json:"previous_30d"` // Score 30 days ago
    Delta      float64 `json:"delta"`        // Current - Previous
    Direction  string  `json:"direction"`    // "rising", "falling", "stable"
}
```

**In enrichment pipeline:**
- After `EnrichBatch`, call `app.EPSS.GetTimeSeries(ctx, cveID, 30)` for each CVE
- Compute delta and direction
- Flag "rising" CVEs in table output with indicator

**Table output:**
```
CVE-2024-24790  CRITICAL  0.87‚Üë  KEV  P0-CRITICAL  [EXPLOIT]
                          (was 0.02 30d ago, +0.85)
```

**Policy integration:**
```yaml
- name: "Block rapidly rising EPSS"
  match:
    epss_trend: rising
    epss_gte: 0.5
  action: fail
```

## Item 12: `vulnex init` Onboarding Wizard

### Implementation

**`cmd/init.go`**:
1. Detect project type by scanning for lockfiles/SBOMs in current directory
2. Run first scan automatically
3. Suggest CI snippet based on detected CI system (.github/workflows, .gitlab-ci.yml)
4. Create starter `.vulnex-policy.yaml` with sensible defaults
5. Print time-to-first-value summary

```bash

Detecting project type... Go (found go.sum)
Running first scan... 12 vulnerabilities found (2 P0, 3 P1, 7 P3)
Detected CI: GitHub Actions

Suggested .github/workflows/vulnex.yml:
  ...

Created .vulnex-policy.yaml with default rules.
Created .vulnexignore (empty).

Setup complete! Run 'vulnex scan go.sum --enrich' to see full results.
```
