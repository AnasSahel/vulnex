Features

Everything you need in one query

Combine five vulnerability data sources, score risk, scan SBOMs, and pipe results — all from a single static binary.

🔍

Multi-Source Enrichment

Combine NVD, KEV, EPSS, GHSA, and OSV data into a single enriched view for any CVE.

⚠

Risk Scoring

P0–P4 composite priority matrix blending CVSS, EPSS, and KEV signals for actionable triage.

📦

SBOM Scanning

Parse CycloneDX/SPDX SBOMs, find vulnerable components, and generate OpenVEX documents.

🔄

SBOM Diffing

Compare two SBOMs and see which vulnerabilities a dependency change introduces or fixes. CI gate built in.

💻

Offline & Pipe-Friendly

Local SQLite cache with configurable TTLs. Stdin support, multiple output formats, composable commands.

piping 
# Recent KEV entries → enrich → JSON
$ vulnex kev recent --days 7 -o csv \
    | cut -d, -f1 \
    | vulnex enrich --stdin -o json

# Bulk EPSS scoring from a file
$ cat cves.txt \
    | vulnex epss score --stdin -o csv \
    > scores.csv

Installation

cd vulnex
make build
CI/CD

Gate your pipeline on vulnerabilities

sbom check catches existing vulns. sbom diff catches new vulns a PR introduces. Both exit 1 on failure.

.github/workflows/security.yml — block new vulnerabilities
name: Vulnerability Diff Gate
on: [pull_request]

jobs:
  vuln-diff:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install vulnex
        run: go install github.com/trustin-tech/vulnex@latest

      - name: Generate SBOMs
        run: |
          git stash && cyclonedx-gomod app > old-bom.json
          git stash pop && cyclonedx-gomod app > new-bom.json

      - name: Diff for new vulnerabilities
        run: vulnex sbom diff old-bom.json new-bom.json
        # Exits 1 if new vulns introduced → blocks the PR
.github/workflows/security.yml — full vulnerability check
name: Security Gate
on: [push, pull_request]

jobs:
  vuln-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install vulnex
        run: go install github.com/trustin-tech/vulnex@latest

      - name: Generate SBOM
        run: cyclonedx-gomod app > bom.json

      - name: Check for vulnerabilities
        run: vulnex sbom check bom.json --severity critical
        # Exits 1 if critical vulns found → fails the job
